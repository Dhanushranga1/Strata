## 💡 Pro Tips

### For Resumes:
- ✅ **Lead with metrics** — "Reduced X by 60%" beats "Improved X"
- ✅ **Use full terms once** — "RAG (Retrieval-Augmented Generation)" for ATS
- ✅ **Match keywords** — Mirror language from job description
- ✅ **Show progression** — From implementation → optimization → leadership

### For Interviews:
- ✅ **STAR but natural** — Don't sound robotic, have a conversation
- ✅ **30% on Results** — Spend most time on measurable outcomes
- ✅ **Have code ready** — Files open in IDE, ready to screenshare
- ✅ **Draw diagrams** — Practice on whiteboard/paper beforehand
- ✅ **Be honest** — Say "approximately" for estimates, "measured" for facts

### For Portfolio:
- ✅ **Show, don't tell** — Screenshots, videos, interactive demos
- ✅ **Open source it** — Public GitHub = instant credibility
- ✅ **Blog about it** — Write up your RAG implementation journey
- ✅ **Link everything** — Resume → Portfolio → GitHub → Gists → LinkedIn

### For Technical Questions:
- ✅ **Have proof ready** — Pull up actual files, run verification commands
- ✅ **Explain trade-offs** — Why you chose X over Y
- ✅ **Admit limitations** — "This is estimated" or "I'd improve X in v2"
- ✅ **Show evolution** — Before optimization → After optimization
