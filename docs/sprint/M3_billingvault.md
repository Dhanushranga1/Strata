## What This Is

BillingVault is a complete invoice lifecycle module: create clients → build invoices with line items → generate branded PDFs → email to clients → track payments. Includes letterhead setup with company logo, bank details, and auto-sequential invoice numbering.

## Schema

```
app.billing_profiles  — org's company info, logo, bank details, invoice numbering
app.billing_clients   — billing clients (separate from platform users)
app.invoices          — invoice headers with status, amounts, dates
app.invoice_items     — line items: description, qty, unit price, discount %, tax %
app.invoice_payments  — payment records (manual + Stripe webhook ready)
```

---

## Invoice Status Flow

```
draft → sent → viewed → paid
              ↓
           partial (part-paid)
              ↓
           overdue (auto-marked when due_date passes)
           
any non-paid state → void
```

## PDF Letterhead Layout

Generated by `billing_pdf.py` using ReportLab Platypus:

1. Company logo (top-left) + Invoice/Status stamp (top-right, color-coded)
2. Divider in brand indigo
3. Three-column block: FROM / BILL TO / Invoice Details
4. Balance-due badge (with overdue color if applicable)
5. Line items table (branded header, alternating rows)
6. Totals: subtotal → discount → tax → total → balance due
7. Notes (if any)
8. Payment history (if partial payments exist)
9. Bank / payment details (from billing profile)
10. Footer + "Generated by Strata BillingVault" attribution

Logo is stored as base64 in `billing_profiles.logo_base64` — rendered inline into the PDF, no filesystem required.

## Auto-Features

- **Invoice numbering** — `{prefix}-{year}-{####}` e.g. `INV-2025-0001`, auto-incremented per org
- **Auto due date** — calculated from client's `payment_terms_days` or org default (30 days)
- **Auto currency** — pre-fills from selected client's preferred currency
- **Auto status transitions** — paying full amount sets status → `paid`; partial payment → `partial`
- **Overdue function** — `app.mark_overdue_invoices()` ready to run via pg_cron or scheduler

## How to Verify

1. Go to `/billing/profile` — fill in company details, upload a logo, add bank details
2. Go to `/billing/clients/new` — create a test client with email
3. Go to `/billing/invoices/new` — select client, add 2-3 line items with different tax rates, verify auto-totals
4. Save draft → Download PDF → verify letterhead, line items, totals, bank details all appear
5. Click "Send to Client" — status → `sent`, email sent if `SENDGRID_API_KEY` set
6. Click "Record Payment" → enter amount → status → `paid` or `partial`
7. Dashboard at `/billing` shows updated stats
8. Platform Hub at `/platform` shows BillingVault card with live stats
