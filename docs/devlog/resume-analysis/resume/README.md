## ✏️ Customization Tips

### For Different Roles:

**Backend Developer:**
```
✅ Use: Backend Engineering + AI/ML + Performance bullets
❌ Avoid: Too much frontend emphasis
```

**Full-Stack Developer:**
```
✅ Use: Mix of backend, frontend, and DevOps bullets
✅ Emphasize: End-to-end ownership
```

**AI/ML Engineer:**
```
✅ Use: AI/ML + Backend + Data & Analytics bullets
✅ Highlight: RAG implementation and optimization
```

**DevOps Engineer:**
```
✅ Use: DevOps + Performance + Backend bullets
✅ Emphasize: CI/CD automation and optimization
```

### Tailoring for ATS:

1. **Match keywords** from job description
2. **Use acronyms AND full terms** (e.g., "RAG (Retrieval-Augmented Generation)")
3. **Include technologies** by name (FastAPI, React, PostgreSQL)
4. **Lead with metrics** (60% reduction, 85% accuracy, etc.)

### Power Verbs by Level:

**Junior/Mid-Level:**
- Implemented, Developed, Built, Created, Designed

**Senior:**
- Architected, Engineered, Optimized, Led, Delivered

**Lead/Principal:**
- Drove, Transformed, Scaled, Established, Pioneered

## 🎨 Formatting Example

### Good Resume Bullet:
```
Engineered production RAG system using FAISS vector store and Google Gemini 
embeddings, achieving 85% citation coverage and 72% average confidence scores 
across 15K+ knowledge base chunks, reducing manual ticket handling by 60%
```

**Why it works:**
- ✅ Strong verb (Engineered)
- ✅ Technical specifics (FAISS, Gemini, RAG)
- ✅ Quantified results (85%, 72%, 15K+, 60%)
- ✅ Business impact (reduced manual handling)
- ✅ One line, ~30 words

### Avoid:
```
❌ Worked on AI features for customer support
```
(Too vague, no metrics, weak verb)
